# 售前闭环智能体平台_飞书接入最小功能清单与开发任务拆解

## 1. 文档目标

本文档用于明确“售前闭环智能体平台”接入飞书的一期最小实现范围，并将前后端、平台层、同步任务、权限与测试工作拆解为可执行任务，供研发分工与排期使用。

本期目标不是做完整 CRM，也不是一次性做飞书单点登录，而是先完成：

1. 智能体结果可人工确认后发送到飞书
2. 平台内自动创建售前跟进任务
3. 任务节点能够通过飞书通知/提醒到责任人
4. 附件与会后产物可归档并在飞书协同传播
5. 飞书用户与部门能够同步到平台，用于任务分派与权限映射
6. 飞书接收人可在卡片内一键转为自己的飞书个人任务

## 2. 一期最小功能清单

### 2.1 需求分析结果流转到飞书

目标：让客户需求分析报告在人工确认后，发送到指定飞书会话或责任人。

最小能力：
- 需求分析报告页新增 `发送到飞书`
- 弹出发送确认框
- 可选择接收人/群组/部门负责人
- 可编辑发送摘要与附言
- 可勾选是否附带：
  - 需求分析报告摘要
  - 推荐追问问题
  - 附件清单
- 发送成功后在平台内留存发送记录

MVP 不做：
- 自动无确认发送
- 飞书长文档自动生成
- 富交互卡片复杂审批流

当前已纳入：
- 飞书交互卡片消息
- 卡片接收人一键转为个人飞书任务
- 卡片成功态回写与幂等控制

### 2.2 解决方案结果流转到飞书

目标：让已生成的解决方案结果在人工确认后，同步给销售、售前、技术支持等协作角色。

最小能力：
- 解决方案结果页新增 `发送到飞书`
- 支持发送：
  - 方案标题
  - 摘要
  - 方案链接
  - PDF/HTML 附件链接或归档链接
- 支持选择发送目标
- 支持记录发送状态与失败原因

### 2.3 售前跟进任务自动创建

目标：当需求分析或方案生成完成后，自动形成平台内待跟进任务。

最小能力：
- 支持从以下事件自动建任务：
  - 需求分析报告完成
  - 解决方案生成完成
  - 用户手动创建任务
- 任务字段最小集合：
  - 任务标题
  - 客户名称
  - 来源会话/来源报告/来源方案
  - 负责人
  - 协同人
  - 截止时间
  - 状态
  - 优先级
  - 备注
- 任务支持状态流转：
  - 待处理
  - 处理中
  - 已完成
  - 已取消

### 2.4 飞书任务通知与回访提醒

目标：把平台内任务动作通过飞书送达责任人，减少人工遗漏。

最小能力：
- 新任务创建后向责任人发送飞书提醒
- 到期前发送提醒
- 状态变更可选通知
- 支持“下一次回访时间”提醒
- 支持成员树按部门展开勾选
- 支持成员与群聊同次发送
- 支持同一飞书账号在多个部门下分别展示并分别勾选

MVP 不做：
- 飞书内直接回写平台任务状态
- 飞书审批工作流深度联动

### 2.5 附件与资料归档

目标：把售前过程中的关键产物沉淀下来，并可在飞书里引用。

最小能力：
- 归档对象：
  - 会议录音源文件
  - 需求分析报告
  - 解决方案 PDF/HTML/Markdown
  - 会后补充附件
- 平台内可查看归档记录
- 飞书消息中可附带归档链接或附件说明

### 2.6 飞书身份与部门同步

目标：保证消息发送对象、任务责任人、部门归属一致。

最小能力：
- 飞书部门同步到平台
- 飞书用户同步到平台
- 平台用户与飞书用户建立映射
- 离职用户自动停用平台账户并触发待转派处理
- 平台账号与飞书同步账号支持人工合并
- 平台部门与飞书同步部门支持人工合并
- 支持飞书部门同步白名单，避免无关组织进入平台

MVP 不做：
- 完整飞书登录替换平台登录
- 全量组织架构双向同步

## 3. 产品与系统边界

### 3.1 飞书承担什么角色

飞书承担：
- 通知送达
- 协同触达
- 轻量提醒
- 任务消息承载
- 团队内部信息扩散

### 3.2 平台承担什么角色

平台承担：
- 主业务逻辑
- 任务主数据
- 归档主数据
- 权限判断
- 审计日志
- 智能体结果生成

### 3.3 本期明确不做的内容

- 完整 CRM 系统
- 客户画像完整体系
- 飞书单点登录替代平台登录
- 企业微信/钉钉并行接入
- 全自动无确认写入飞书和外部系统

## 5. 后端开发任务拆解

### 5.1 账户与部门同步

后端任务：
1. 新增飞书账户映射字段
2. 新增飞书部门映射字段
3. 新增飞书同步服务
4. 新增全量同步命令
5. 新增增量同步命令
6. 增加离职用户停用逻辑
7. 增加待转派任务查询能力
8. 新增平台账号与飞书账号合并能力
9. 新增平台部门与飞书部门合并能力
10. 支持部门白名单同步

验收标准：
- 可同步部门树
- 可同步用户基础信息
- 可识别离职状态并停用账户

### 5.2 任务中心

后端任务：
1. 建立 `PresalesTask` 模型与迁移
2. 建立任务列表/详情/编辑/状态变更接口
3. 建立自动建任务服务
4. 支持来源对象回链
5. 支持截止时间与回访提醒字段

验收标准：
- 从需求分析报告完成后可自动建任务
- 从解决方案完成后可自动建任务

### 5.3 归档中心

后端任务：
1. 建立 `PresalesArchiveRecord` 模型与迁移
2. 建立归档记录创建接口
3. 支持从现有录音、报告、方案结果自动生成归档记录
4. 支持下载与删除权限控制

验收标准：
- 录音、报告、方案结果均可形成归档条目

### 5.4 飞书消息投递

后端任务：
1. 建立飞书应用配置读取逻辑
2. 建立飞书 token 获取与缓存逻辑
3. 建立消息发送服务
4. 建立发送记录落库逻辑
5. 建立失败重试与错误回写
6. 升级为飞书交互卡片发送
7. 支持多成员与多群聊同次发送
8. 支持按部门展开成员树收件人接口

验收标准：
- 平台可向指定用户发送消息
- 平台可向指定群组发送消息
- 发送成功/失败均可审计

### 5.5 提醒任务

后端任务：
1. 建立到期提醒扫描任务
2. 建立回访提醒扫描任务
3. 支持触发飞书通知

验收标准：
- 到期前提醒可送达
- 回访提醒可送达

### 5.6 飞书个人任务

后端任务：
1. 新增飞书个人任务授权接口
2. 保存个人授权状态与授权时间
3. 新增卡片交互回调接口
4. 新增 `FeishuTaskRecord` 留痕
5. 按飞书用户身份创建个人任务
6. 增加同人同任务幂等控制
7. 卡片成功态回写
8. 群卡片首击异步创建，避免超时报错

验收标准：
- 已授权接收人可在飞书卡片中一键创建自己的飞书任务
- 同一群内不同成员可分别创建自己的飞书任务
- 重复点击不会重复建任务

## 9. 推荐实施顺序

### 第一阶段：打通最小协同闭环

1. 飞书身份映射字段
2. 飞书用户/部门同步
3. 平台内任务模型与接口
4. 需求分析报告发送到飞书
5. 解决方案结果发送到飞书
6. 自动建任务 + 飞书提醒

### 第二阶段：归档与提醒增强

1. 资料归档中心
2. 回访提醒
3. 发送记录与发送状态页
4. 管理配置页

### 第三阶段：后续扩展

1. 飞书登录/单点登录评估
2. 更复杂的飞书卡片交互
3. CRM 主档与客户画像衔接
4. 企业微信/钉钉二期评估

## 10. 一期 MVP 验收标准

满足以下条件即可认为一期飞书接入达成：

1. 需求分析报告可人工确认后发送到飞书
2. 解决方案结果可人工确认后发送到飞书
3. 平台能够自动生成售前跟进任务
4. 飞书能够向责任人发送任务提醒
5. 飞书用户与部门可同步到平台
6. 离职用户会被停用且不再继续接收新任务提醒
7. 录音、报告、方案结果可形成平台归档记录

## 11. 当前建议

当前最适合的落地策略是：

- `飞书做协同层`
- `平台做主业务层`
- `用户与部门由飞书同步`
- `角色与权限由平台维护`

先把“需求分析 -> 方案 -> 任务 -> 提醒 -> 归档”的内部闭环跑通，再考虑 CRM 和客户画像扩展。
