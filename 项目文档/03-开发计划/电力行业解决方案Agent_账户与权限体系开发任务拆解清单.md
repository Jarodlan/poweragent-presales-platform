# 电力行业解决方案Agent 账户与权限体系开发任务拆解清单

## 1. 文档目标

本清单用于把“账户-角色-权限-部门”体系拆成可分配、可估时、可验收的开发任务，便于后端、前端、测试并行推进。

## 2. 范围说明

本清单覆盖：

- Django 平台层账户体系
- 认证与鉴权
- 角色权限与部门
- 超级管理员初始化
- 会话/任务数据权限
- 前端登录与管理页面基础能力

不覆盖：

- SSO / LDAP 正式接入
- 多租户
- 外部注册

## 3. 阶段划分

建议拆成 5 个阶段：

1. `阶段A：数据模型与初始化`
2. `阶段B：认证与当前用户能力`
3. `阶段C：用户、角色、部门后台能力`
4. `阶段D：会话与任务资源权限`
5. `阶段E：前端登录与管理页面`

## 4. 后端开发任务

## 4.1 阶段A：数据模型与初始化

### A1. 扩展 User 模型

任务内容：

- 在 `accounts.User` 增加业务字段
- 保持兼容 Django `AbstractUser`

交付物：

- 模型代码
- migration

验收标准：

- 可成功迁移
- Django Admin 可查看新字段

### A2. 新增 Department 模型

任务内容：

- 支持部门基本信息与 parent 结构

交付物：

- 模型代码
- migration
- admin 注册

### A3. 新增 Role / PermissionCode / UserRole / RolePermission 模型

任务内容：

- 定义角色、权限、关系表
- 补唯一约束

交付物：

- 模型代码
- migration
- admin 注册

### A4. 实现 `bootstrap_rbac` 管理命令

任务内容：

- 初始化系统角色
- 初始化权限码
- 初始化角色权限关系

交付物：

- management command
- 执行说明

验收标准：

- 重复执行幂等
- 不产生重复角色和权限

### A5. 实现 `bootstrap_super_admin` 管理命令

任务内容：

- 创建超级管理员
- 自动分配 `super_admin`

交付物：

- management command

验收标准：

- 可通过脚本创建首个管理员
- 支持幂等校验

## 4.2 阶段B：认证与当前用户能力

### B1. 接入 JWT 认证

任务内容：

- 集成 SimpleJWT
- 配置 access / refresh token

交付物：

- settings 配置
- 认证依赖
- token 接口

### B2. 登录接口

任务内容：

- 支持 `username/employee_no + password`
- 校验账户状态

交付物：

- `POST /api/v1/auth/login`

验收标准：

- 正常登录返回 token 和用户信息
- disabled/locked 账户不可登录

### B3. 刷新与登出接口

任务内容：

- refresh token
- token 吊销或黑名单预留

交付物：

- `POST /api/v1/auth/refresh`
- `POST /api/v1/auth/logout`

### B4. 当前用户接口

任务内容：

- 返回用户基本信息
- 返回角色码、权限码、部门信息

交付物：

- `GET /api/v1/auth/me`

## 4.3 阶段C：用户、角色、部门后台能力

### C1. 用户管理接口

任务内容：

- 用户列表
- 创建用户
- 编辑用户
- 启用/停用
- 重置密码

交付物：

- `/api/v1/users` 相关接口

### C2. 部门管理接口

任务内容：

- 部门列表
- 新增/编辑部门
- 部门树接口

交付物：

- `/api/v1/departments`
- `/api/v1/departments/tree`

### C3. 角色管理接口

任务内容：

- 查询角色
- 查询权限
- 分配用户角色
- 配置角色权限

交付物：

- `/api/v1/roles`
- `/api/v1/permissions`

### C4. 审计日志增强

任务内容：

- 记录账户相关操作
- 增加 result_status/ip/user_agent 等字段

交付物：

- AuditLog 字段增强
- 审计写入工具函数

## 4.4 阶段D：会话与任务资源权限

### D1. Conversation 增加 owner/department/visibility

任务内容：

- 扩展模型
- 创建会话时自动写入 owner/department

### D2. Task 增加 owner/department

任务内容：

- 扩展模型
- 创建任务时自动写入 owner/department

### D3. 查询接口按数据范围过滤

任务内容：

- 普通用户只能看自己的
- 部门管理员能看本部门
- 平台管理员可看全局

### D4. 资源级权限工具函数

任务内容：

- 实现 `can_access_conversation`
- 实现 `can_access_task`
- 应用于视图层

## 5. 前端开发任务

## 5.1 登录与认证

### F1. 登录页

任务内容：

- 登录表单
- 错误提示
- 登录成功跳转

### F2. 用户状态 Store

任务内容：

- 保存 token

- 提供 `hasPermission()`

### F3. 路由守卫

任务内容：

- 未登录跳转登录页
- 无权限页面拦截

## 5.2 管理页面

### F4. 用户管理页

任务内容：

- 列表、搜索、创建、编辑、停用、重置密码

### F5. 部门管理页

任务内容：

- 部门树展示
- 新增、编辑、成员查看

### F6. 角色权限页

任务内容：

- 角色列表
- 权限点查看
- 角色配置

### F7. 个人中心页

任务内容：

- 个人资料
- 修改密码
- 查看所属部门与角色

- 用户模型 migration 测试
- 管理命令幂等测试

- disabled/locked 账户测试

## 6.2 前端测试

- 登录流程测试
- token 过期跳转测试

- 用户管理操作测试
- 部门管理测试

- 普通员工只能查看自己的会话和任务

## 8. 优先级建议

### P0

- User 扩展
- Department / Role / Permission 模型
- bootstrap_rbac
- bootstrap_super_admin
- 登录接口
- 当前用户接口
- 会话/任务 owner 控制

### P1

- 用户管理
- 部门管理
- 角色管理
- 前端登录与用户管理页面

### P2

- 细粒度审计
- 更复杂数据范围
- SSO / LDAP 预留增强

## 9. 验收清单

交付完成时至少满足：

1. 可通过脚本创建首个超级管理员
2. 可通过后台创建普通内部用户
3. 用户可登录并获取个人角色和权限
4. 会话和任务具备 owner/department 控制
5. 部门管理员无法跨部门管理账户
6. 普通员工无法查看他人会话
7. 所有关键账户操作可追溯到审计日志

## 10. 推荐推进顺序

最稳的推进方式是：

1. 先做模型和初始化命令
2. 再做认证
3. 再做管理接口
4. 再做资源访问控制
5. 最后接前端页面

一句话总结：

`先把身份、角色、归属和初始化能力打牢，再把会话/任务纳入权限控制，最后补前端管理能力。`
