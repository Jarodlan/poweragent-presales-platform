# 客户需求分析智能体 接口设计文档

## 1. 文档目的

本文档定义“客户需求分析智能体”MVP 阶段的对外接口与内部服务接口，供前端、Django 平台层、Agent Service、ASR 接入层协同开发使用。

设计目标：

1. 接口语义清晰
2. 支持会话式工作流
3. 支持实时记录与分析任务分离
4. 支持知识库开关
5. 与现有 PowerAgent 接口风格保持一致

---

## 2. 接口分层

### 2.1 前端 -> Django 平台层

负责：

1. 会话管理
2. 录音控制
3. 历史查看
4. 分析任务触发
5. 结果查询

### 2.2 前端 -> 流式层

负责：

1. ASR 文本流订阅
2. 会话状态流订阅

说明：

1. 前端不直接区分底层是 `Qwen` 还是 `FunASR`
2. 前台统一对接 Django 平台层的上传接口与流式订阅接口
3. Django 平台层内部通过 `ASR Adapter` 路由到具体提供方

### 2.3 Django 平台层 -> Agent Service

负责：

1. 阶段整理任务
2. 最终需求分析任务
3. 结果回调

### 2.4 Agent Service -> RAGFlow

仅在启用知识库时使用。

## 5. 音频与流式接口

## 5.1 上传音频分片

### 接口

`POST /api/v1/customer-demand/sessions/{session_id}/segments/audio`

### Content-Type

`multipart/form-data`

### 表单字段建议

| 字段 | 类型 | 说明 |
|---|---|---|
| `audio_chunk` | file | 音频分片 |
| `chunk_index` | int | 分片序号 |
| `started_at_ms` | bigint | 起始毫秒 |
| `ended_at_ms` | bigint | 结束毫秒 |
| `provider` | string | 可选，`qwen` / `funasr` |
| `language` | string | 可选，默认 `zh` |
| `audio_fs` | int | 可选，默认 `16000` |
| `mock_text` | string | 可选，本地联调用占位文本 |

### 返回

```json
{
  "code": 0,
  "message": "ok",
  "data": {
    "accepted": true,
    "chunk_index": 12,
    "segment": {},
    "adapter_metadata": {
      "provider": "qwen_realtime_asr"
    }
  }
}
```

## 7. 知识库接口使用策略

### 7.1 启用知识库时

Agent Service 内部按如下逻辑：

1. 判断行业相关性
2. 构造检索 query
3. 检索 RAGFlow
4. 只将高相关结果作为补充证据

### 7.2 不启用知识库时

1. 不发起检索请求
2. 完全基于会话文本分析

## 8. ASR Adapter 内部设计说明

### 8.1 目标

在 Django 平台层增加统一的 ASR 抽象层，让业务接口不与具体厂商能力强耦合。

### 8.2 当前规划的适配器

1. `QwenAsrAdapter`
2. `FunAsrAdapter`

### 8.3 统一调用方式

建议统一内部方法：

`transcribe_audio_chunk(audio_bytes, provider, sample_rate, language, metadata)`

### 8.4 推荐策略

1. 默认 `Qwen`
2. `FunASR` 作为可替换实现
3. 保留 `mock_text` 便于本地联调与前后端并行开发

### 8.5 当前 MVP 实现说明

当前 MVP 已优先接入 `QwenAsrAdapter` 的真实 HTTP 调用能力，默认使用：

1. `model = qwen3-asr-flash`
2. `base_url = https://dashscope.aliyuncs.com/compatible-mode/v1`
3. `POST /chat/completions` 的 OpenAI compatible 调用方式

当前适用范围：

1. 浏览器或移动端上传的短音频分片
2. 单段音频大小控制在官方建议范围内
3. 通过 Base64 Data URL 发送音频内容

后续扩展方向：

1. 升级到 `Qwen3-ASR-Flash-Realtime` WebSocket 实时流式模式
2. 接入 `FunASR` 的 2pass / 流式实现

## 10. 错误码建议

| code | 含义 |
|---|---|
| `45001` | 会话不存在 |
| `45002` | 会话状态不允许当前操作 |
| `45003` | 录音尚未开始 |
| `45004` | 录音已结束 |
| `45005` | 分析任务已存在 |
| `45006` | 音频上传失败 |
| `45007` | ASR 调用失败 |
| `45008` | 语义校验失败 |
| `45009` | 报告生成失败 |

## 11. 推荐开发顺序

### 第一步

1. 会话 CRUD
2. 分段存储
3. 基础分析任务状态

### 第二步

1. ASR 分段上传
2. 流式文本展示
3. 阶段整理接口

### 第三步

1. 最终报告接口
2. 知识库开关
3. 导出能力

---

## 12. 结论

客户需求分析智能体接口设计应遵循：

1. 前台会话式接口
2. 后台任务式接口
3. 流式事件与最终结果分离
4. 知识库增强可选

建议先将“记录 -> 分段 -> 分析 -> 输出”四条链路打通，再逐步丰富导出、联动、自动补问等增强能力。
