# 电力行业解决方案生成Agent Vue前端开发任务拆解清单

## 1. 文档目标

本文档面向前端工程师和全栈工程师，目标是把当前已有 PRD、技术架构、接口文档、页面原型说明，进一步拆解成可直接排期和开发的 Vue 前端任务清单。

适用范围：

- `Vue 3 + Vite + Element Plus + Pinia`
- ChatGPT 风格会话工作台
- 历史会话列表
- 流式方案生成
- 证据卡展示

## 2. 开发目标

前端需完成一个可用的内部 Demo 页面，满足以下能力：

1. 支持新建会话
2. 支持会话列表展示
3. 支持加载历史会话
4. 支持在当前会话中继续提问
5. 支持流式展示 Agent 状态与内容
6. 支持渲染摘要、正文和证据卡
7. 支持停止生成、复制摘要、复制全文

## 3. 当前前端开发所需文档是否足够

结论：

`已经基本满足前端工程师开工所需。`

当前已具备：

- PRD
- 技术架构说明
- 接口文档
- 前端页面详细原型说明

前端开发当前不缺“方向性文档”，已经可以进入：

- 页面搭建
- 状态管理
- 接口联调
- SSE 接入

因此，本清单默认前端可以直接开工。

## 5. 任务拆解总览

| 模块 | 任务 | 优先级 |
|---|---|---|
| 工程初始化 | 项目结构、路由、样式基线 | P0 |
| 页面骨架 | ChatGPT 风格工作台布局 | P0 |
| 会话侧边栏 | 会话列表、新建、切换、分组 | P0 |
| 消息流区域 | 用户消息、助手消息、状态消息 | P0 |
| 输入区 | 输入框、参数折叠面板、发送/停止 | P0 |

## 6. 按阶段拆解

## 6.1 阶段一：工程初始化

### 任务 1：初始化 Vue 工程

内容：

- 初始化 `Vue 3 + Vite`
- 安装 `Element Plus`
- 安装 `Pinia`
- 安装 `vue-router`
- 安装 `markdown-it`

交付标准：

- 本地可启动
- 有基础路由
- 有全局样式入口

### 任务 2：建立目录结构

内容：

- 创建 `api/`
- 创建 `components/`
- 创建 `stores/`
- 创建 `types/`
- 创建 `utils/`

交付标准：

- 目录结构与说明书保持一致
- 有基础空文件占位

### 任务 3：建立基础样式系统

内容：

- 定义颜色变量
- 定义间距变量
- 定义卡片、侧边栏、输入区基础样式

交付标准：

- 页面具备统一视觉基线
- 不出现默认浏览器风格

## 6.2 阶段二：页面骨架

### 任务 4：实现 Workspace 主页面

组件：

- `WorkspaceView.vue`

内容：

- 左侧侧边栏
- 中间消息工作区
- 底部输入区
- 右侧证据抽屉占位

交付标准：

- 页面结构完整
- 适配 1440 宽桌面端

### 任务 5：实现响应式布局

内容：

- 桌面端三段布局
- 小屏隐藏侧边栏为抽屉

交付标准：

- 宽屏正常
- 1366 宽度正常
- 移动端不破版

## 6.3 阶段三：会话侧边栏

### 任务 6：会话列表组件

组件：

- `ConversationSidebar.vue`
- `ConversationList.vue`
- `ConversationItem.vue`

内容：

- 列表分组展示
- 当前会话高亮
- 会话标题、更新时间、最近提问展示

接口依赖：

- `GET /api/v1/conversations`

交付标准：

- 能展示会话列表
- 空列表有空态

### 任务 7：新建会话

内容：

- 点击新建会话调用创建接口
- 新会话插入列表顶部
- 自动选中并清空消息区

接口依赖：

- `POST /api/v1/conversations`

交付标准：

- 新会话可立即进入输入态

### 任务 8：切换会话与加载历史

内容：

- 点击会话后获取会话详情
- 拉取历史消息
- 渲染消息区

接口依赖：

- `GET /api/v1/conversations/{conversation_id}`
- `GET /api/v1/conversations/{conversation_id}/messages`

交付标准：

- 切换会话后消息区刷新正确
- 历史消息顺序正确

## 6.4 阶段四：消息流区域

### 任务 9：实现用户消息组件

组件：

- `UserMessageBubble.vue`

内容：

- 右对齐气泡
- 支持多行文本

交付标准：

- 样式接近聊天产品

### 任务 10：实现助手消息组件

组件：

- `AssistantMessageCard.vue`

内容：

- 左对齐
- 支持摘要、正文、假设条件、证据入口

交付标准：

- 长文渲染稳定
- 有明显内容层级

### 任务 11：实现状态消息组件

组件：

- `StatusMessageBar.vue`

内容：

- 展示当前步骤
- 支持进度或文案变更

交付标准：

- SSE 状态更新时可实时变化

## 6.5 阶段五：输入区与参数面板

### 任务 12：实现输入框

组件：

- `MessageComposer.vue`

内容：

- 多行输入
- Enter 发送
- Shift + Enter 换行

交付标准：

- 输入体验顺滑

### 任务 13：实现参数折叠面板

组件：

- `ParamConfigPanel.vue`

内容：

- 电网场景
- 诊断对象
- 数据基础
- 目标能力

接口依赖：

- `GET /api/v1/meta/options`

交付标准：

- 可读取默认值
- 可修改后随消息提交

### 任务 14：发送消息

内容：

- 当前会话中发送消息
- 本地先插入用户消息
- 创建 assistant 占位消息

接口依赖：

- `POST /api/v1/conversations/{conversation_id}/messages`

交付标准：

- 点击发送后页面立即有响应

### 任务 15：停止生成

内容：

- 生成中展示停止按钮
- 点击后取消任务

接口依赖：

- `POST /api/v1/solution/tasks/{task_id}/cancel`

交付标准：

- 停止后状态终止
- 页面不崩溃

## 6.6 阶段六：SSE 流式接入

### 任务 16：封装 SSE 客户端

文件建议：

- `utils/sse.ts`

内容：

- 打开 SSE
- 订阅事件
- 自动关闭
- 错误回调

交付标准：

- 可稳定接收服务端事件

### 任务 17：处理流式事件

需处理：

- `status`
- `conversation_meta`
- `message_created`
- `summary_chunk`
- `content_chunk`
- `evidence_cards`
- `completed`
- `error`

交付标准：

- 不同事件能驱动界面正确更新

### 任务 18：流式内容拼接

内容：

- 摘要按 chunk 追加
- 正文按 chunk 追加
- 结束时合并为完整 assistant 消息

交付标准：

- 不出现内容错位
- 不出现重复拼接

## 6.7 阶段七：结果区与证据卡

### 任务 19：摘要卡组件

组件：

- `SolutionSummaryCard.vue`

内容：

- 展示摘要
- 展示假设条件

### 任务 20：正文渲染组件

组件：

- `SolutionMarkdownView.vue`

内容：

- 渲染 Markdown
- 支持标题、列表、表格

交付标准：

- 电力方案正文可读性良好

### 任务 21：证据卡列表

组件：

- `EvidenceCardList.vue`
- `EvidenceCard.vue`

内容：

- 展示来源类型
- 展示标题、摘要、章节关联

交付标准：

- 一次生成结果至少可渲染多张证据卡

### 任务 22：证据抽屉

组件：

- `EvidenceDrawer.vue`

内容：

- 点击证据卡打开
- 展示更多细节

交付标准：

- 抽屉开关流畅

## 6.8 阶段八：状态管理

### 任务 23：conversationStore

内容：

- 会话列表

- 加载状态

### 任务 24：messageStore

内容：

- 当前消息列表

### 任务 25：composerStore

### 任务 26：taskStore

## 6.9 阶段九：体验与稳定性

### 任务 28：骨架屏与加载态

内容：

- 消息区骨架
- 长文加载过渡

### 任务 29：错误态

内容：

- SSE 断开
- 接口错误

### 任务 30：复制能力

内容：

- 复制摘要

### 任务 31：滚动体验

内容：
